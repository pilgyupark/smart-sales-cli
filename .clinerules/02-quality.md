# 코드 품질 규칙

## 1. 기술 제약
- Python 표준 라이브러리만 사용한다.
- 외부 패키지를 설치하거나 새로운 프레임워크를 추가하지 않는다.
- 저장 데이터의 기존 JSON 필드명과 데이터 구조를 임의로 변경하지 않는다.
- 새로운 JSON 필드, 상태값, 파일을 추가해야 한다면 먼저 필요성을 설명하고 승인을 기다린다.

## 2. 보안과 입력 검증
- API Key, 비밀번호, 토큰, 개인정보를 코드나 샘플 데이터에 하드코딩하지 않는다.
- 외부 입력값은 형식, 필수 여부, 허용 범위, 공백 입력을 검증한다.
- 날짜 입력값은 요구사항에 정의된 형식을 따른다. 별도 정의가 없다면 `YYYY-MM-DD` 형식을 사용한다.
- 잘못된 날짜 형식과 실제로 존재하지 않는 날짜는 차단한다.
- 오류 메시지는 원인을 이해할 수 있도록 작성하되 민감정보를 노출하지 않는다.

## 3. 데이터 보존과 예외 처리
- JSON 저장 시 기존 데이터가 의도하지 않게 삭제되거나 덮어써지지 않도록 한다.
- 신규 데이터 추가, 기존 데이터 수정, 데이터 삭제 동작을 명확히 구분한다.
- 잘못된 입력, 빈 파일, 파일 누락, 잘못된 JSON 형식이 발생해도 프로그램 전체가 비정상 종료되지 않도록 처리한다.
- 파일 저장 중 오류가 발생하면 원인을 이해할 수 있는 메시지를 제공한다.

## 4. 테스트
- 새 기능을 추가하거나 기존 기능을 수정하면 관련 unittest를 함께 작성하거나 보완한다.
- 정상 입력뿐 아니라 경계값, 빈 문자열, 공백 문자열, 중복 입력, 잘못된 날짜 형식, 잘못된 상태 전이를 테스트한다.
- 기존 기능이 훼손되지 않았는지 전체 unittest를 실행한다.
- 테스트 파일명은 `tests/test_<기능명>.py` 형식을 사용한다.
- 테스트 메서드명은 검증 목적을 알 수 있도록 작성한다.

## 5. 변경 최소화
- 요청한 기능 구현에 필요한 최소 범위만 변경한다.
- 불필요한 리팩터링, 포맷 변경, 주석 대량 수정, 파일 이동을 하지 않는다.
- 기존 함수명과 공개 인터페이스는 가능한 한 유지한다.
