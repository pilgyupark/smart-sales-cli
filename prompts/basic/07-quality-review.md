# 07. 품질 점검과 문서화 검토

```text
현재 프로젝트를 전체 점검해줘.

[중요]
- 파일을 생성하거나 수정하지 마.
- 터미널에서 수정 명령을 실행하지 마.
- 보고서만 작성해줘.

[점검 항목]
1. REQUIREMENTS.md 대비 누락 기능
2. 입력값 검증 누락
3. 테스트 부족 영역
4. 불필요한 리팩터링 여부
5. 기존 JSON 필드명 변경 여부
6. 허용되지 않은 외부 패키지 추가 여부
7. README.md에 추가해야 할 실행 방법
8. 수정이 필요한 파일 목록
9. 수정 우선순위

각 문제를 다음 형식의 표로 정리해줘.
- 문제 내용
- 영향 범위
- 수정이 필요한 파일
- 권장 수정 방법
- 우선순위

아직 코드는 수정하지 마.
```
